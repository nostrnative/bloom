# Nostr Relay Integration Plan

This document outlines the strategy for integrating a parallel Nostr relay into the Blossom Server application using the `nostrnative` plugin as the shared logic layer.

## Architecture
- **`nostrnative` (Plugin)**: Core relay logic, event validation, NIP implementations, and LMDB storage management.
- **`blossom` (App)**: UI for configuration, lifecycle management (starting/stopping), and persistent settings.

## Detailed Steps

### Phase 1: `nostrnative` Backend (Shared Logic)
- [x] **1.1 Dependencies**: Added `nostr-relay-builder`, `nostr-database`, and `nostr-lmdb`.
- [x] **1.2 Relay Implementation**: Implemented the `Relay` logic in `src/relay.rs`.
    - Support NIP-01, 09, 11, 40 via `nostr-relay-builder`.
    - Implemented event persistence using LMDB.
- [x] **1.3 Tauri Commands**: Created commands in `src/commands.rs`.
    - `start_nostr_relay(port, path)`
    - `stop_nostr_relay()`
    - `get_nostr_relay_status()`
- [x] **1.4 Permissions**: Added Tauri 2 permission files for relay commands in `permissions/autogenerated/commands/`.

### Phase 2: `blossom` Backend (Lifecycle)
- [x] **2.1 Initialization**: Updated `nostrnative` plugin `init` to start the relay automatically on port 4869 using LMDB storage.
- [x] **2.2 Data Management**: Relay database is stored in `app_local_data_dir()/relay`.

### Phase 3: `blossom` Frontend (UI/UX)
- [x] **3.1 Store Updates**: Added `relayEnabled` and `relayPort` to Zustand store.
- [x] **3.2 Dedicated Relay Page**: Created `src/components/NostrRelay.tsx` with full controls and status.
- [x] **3.3 Navigation**: Integrated "Relay" into the main app sidebar/navigation in `App.tsx`.

## Files Created/Modified
- `../nostrnative/src/relay.rs`: Core relay logic.
- `../nostrnative/src/commands.rs`: Command handlers.
- `../nostrnative/src/lib.rs`: Plugin initialization and handler registration.
- `../nostrnative/permissions/autogenerated/commands/*.toml`: Permission definitions.
- `src/components/NostrRelay.tsx`: Dedicated UI page.
- `src/app/App.tsx`: Navigation updates.
